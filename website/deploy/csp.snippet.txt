Add this line inside the existing `map $server_name $csp_policy { ... }` block
in the TLS-front proxy's /usr/local/etc/nginx/nginx.conf (next to your main site row):

    "iptv.dope.rs" "default-src 'self'; img-src 'self' data:; style-src 'self'; script-src 'self'; connect-src 'self'; font-src 'self'; object-src 'none'; base-uri 'none'; form-action 'none';";

Why this is safe AND sufficient for the site:
  - default-src 'self'      everything same-origin by default
  - img-src 'self' data:    screenshots + the SVG favicon (data: covers inline SVG use)
  - style-src 'self'        external style.css only (no inline styles -> no 'unsafe-inline')
  - script-src 'self'       external app.js only; the JSON-LD block is a non-executable
                            data block and is NOT affected by script-src
  - connect-src 'self'      no client-side fetches at all (downloads are same-origin <a> links)
  - object/base/form locked down

The global add_header Content-Security-Policy $csp_policy already applies it, so no
per-server add_header is needed on iptv.dope.rs.
